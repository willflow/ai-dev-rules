# AI开发规则集

一套用于AI辅助软件开发的系统化规则和最佳实践指南。基于OODA循环方法论，强调简洁、实用和增量式开发的编程哲学。

## 📋 项目概述

本项目提供了一套完整的AI开发指导框架，包括：

- **核心开发哲学** - 以简洁性和实用性为导向的编程理念
- **技术实施标准** - 具体的代码质量、架构和工程实践规范  
- **前端开发规则** - 针对现代前端框架的专门化指导
- **OODA方法论智能体** - 基于观察-定向-决策-行动的系统化问题解决框架

## 🎯 核心理念

### 开发哲学
- **增量进步优于大爆炸** - 小步快跑，确保每次提交都能编译和通过测试
- **从现有代码学习** - 先研究后实施，遵循项目既有模式
- **实用主义胜过教条主义** - 适应项目现实，选择合适的解决方案
- **清晰意图胜过聪明代码** - 选择显而易见的方案，避免炫技

### 简洁性原则
- 单一职责 - 每个函数/类只做一件事
- 避免过早抽象 - 等到真正需要时再抽象
- 拒绝聪明技巧 - 选择无聊但可靠的解决方案
- 如果需要解释，说明太复杂了

### OODA循环智能体

基于军事战略中的OODA循环(Observe-Orient-Decide-Act)，为复杂问题提供系统化解决框架：

#### [`agents/observe.md`](./agents/observe.md) - 观察阶段
**信息收集专家**
- 🔍 系统性收集所有相关数据
- 📝 发现上下文和模式识别
- 📋 全面覆盖，确保无遗漏
- 🎯 纯观察，不做判断

#### [`agents/orient.md`](./agents/orient.md) - 定向阶段  
**分析理解专家**
- 🧠 上下文分析和模式综合
- 🔗 关系映射和依赖理解
- ⚖️ 优先级评估和假设识别
- 🎯 提供洞察，不做决策

#### [`agents/decide.md`](./agents/decide.md) - 决策阶段
**方案评估专家**
- 💡 生成多种可行方案
- ⚖️ 权衡分析和风险评估
- 🎯 可行性评估和推荐形成
- 📋 明确的实施策略

#### [`agents/act.md`](./agents/act.md) - 行动阶段
**精确执行专家**
- ⚡ 精确实施和代码质量保证
- 🧪 全面测试和验证结果
- 📚 文档更新和质量检查
- ✅ 确认完成度

#### [`agents/simplify.md`](./agents/simplify.md) - 简化智能体
**复杂度控制专家**
- 🔍 过度工程检测
- 🎯 需求对齐验证
- 🧹 样板代码清理
- ⚡ 开发者体验优化

## 🚀 使用指南

### 基础开发流程

1. **项目初始化**
   - 研究现有代码库模式
   - 确定技术栈和约定
   - 建立质量检查流程

2. **功能开发**
   - 使用OODA循环进行复杂问题分析
   - 遵循增量开发原则
   - 应用对应的领域规则(前端/后端)

3. **质量保证**
   - 运行代码检查工具
   - 进行功能验证
   - 使用simplify智能体检查过度工程

### OODA循环应用示例

```bash
# 复杂问题解决流程
1. observe    # 收集所有相关信息
2. orient     # 分析和理解上下文  
3. decide     # 评估方案并做决策
4. act        # 精确执行解决方案
5. simplify   # 检查并简化实现
```

### 前端项目特殊流程

```javascript
// 组件开发优先级
标准HTML/框架组件 > 扩展现有组件 > 创建自定义组件

// Hook设计原则
完整封装相关逻辑 > 分散式设计

// 路径规则
跨目录: @/path/to/file
同级: ./relative/path
```
